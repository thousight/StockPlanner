# 12-01: Redis Infrastructure & Lifespan Setup

Provision Redis connectivity and integrate a circuit breaker to manage application availability based on Redis health.

## Tasks

<task id="1">
   <name>Update Dependencies & Configuration</name>
   <files>
     - requirements.txt
     - src/config.py
   </files>
   <action>
     - Add `redis`, `langgraph-checkpoint-redis`, and `aiobreaker` to `requirements.txt`.
     - Update `src/config.py`:
       - Add `REDIS_URL: str = "redis://localhost:6379"`.
       - Add `REDIS_CHECKPOINT_TTL_MIN: int = 30`.
   </action>
   <verify>
     - Check `src/config.py` for new settings.
     - `pip install -r requirements.txt` succeeds.
   </verify>
   <done>
     - [ ] Dependencies and config updated.
   </done>
</task>

<task id="2">
   <name>Implement Redis Lifespan & Circuit Breaker</name>
   <files>
     - main.py
   </files>
   <action>
     - Update `main.py`:
       - Import `redis.asyncio` and `aiobreaker`.
       - Define a global `CircuitBreaker` (`redis_cb`) for Redis operations.
       - Update `lifespan` to initialize a global Redis client stored in `app.state.redis`.
       - Use `redis_cb.call_async` or similar to protect Redis initialization and health checks.
   </action>
   <verify>
     - Application starts without errors when Redis is running.
   </verify>
   <done>
     - [ ] Redis lifespan management implemented.
   </done>
</task>

<task id="3">
   <name>Enforce 503 Availability</name>
   <files>
     - main.py
   </files>
   <action>
     - Add a custom exception handler or middleware in `main.py` that catches both `aiobreaker.CircuitBreakerError` and `redis.exceptions.ConnectionError`.
     - Return `JSONResponse(status_code=503, content={"detail": "Service Temporarily Unavailable: Redis Connection Failure"})`.
   </action>
   <verify>
     - Stopping local Redis results in 503 responses for chat-related dependencies.
   </verify>
   <done>
     - [ ] 503 error handling verified.
   </done>
</task>

## Success Criteria
- Redis client correctly initialized in FastAPI lifespan.
- Application gracefully handles Redis downtime with 503 errors.
- Configuration parameters are externalized to environment variables.
