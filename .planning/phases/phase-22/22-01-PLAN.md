## Verification Criteria
- [ ] `CodeGeneratorAgent` consistently produces code using the `run(input_data)` pattern.
- [ ] The agent successfully fixes its own code after a simulated execution failure.
- [ ] Forbidden code patterns (detected by AST) trigger a correction attempt instead of immediate failure.
- [ ] The agent explains its logic in an "Audit" section before outputting code.

## must_haves
- Maximum of 3 retries for any single request.
- All errors passed back to the agent for correction must be sanitized (no system paths).
- The agent must use vectorized operations (Pandas/NumPy) where appropriate.
