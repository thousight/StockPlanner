# Phase 20: Market Narrative & Sentiment Change Agent

## Step 20-03: Narrative Synthesis Tool

### Goal
Implement the `get_market_narrative` tool that synthesizes current market data and identifies shifts in the narrative.

### Context
This tool performs the heavy lifting: price action lookup (yfinance), news aggregation (web_search), historical lookup (ResearchCache), and shift analysis via LLM.

### Tasks
- [x] Create `src/graph/tools/narrative.py`.
- [x] Implement `get_market_narrative` tool:
    - [x] Fetch Price Action: SPY, QQQ, DIA, VIX via `src/services/market_data.py`.
    - [x] Fetch Professional Recap: Use `web_search` with professional domain filters (Bloomberg/WSJ/CNBC).
    - [x] Identify Previous Session: Use `get_previous_trading_day` from Step 20-02.
    - [x] Retrieve Previous Narrative: Lookup `narrative_YYYYMMDD` in `ResearchCache` (lookback up to 7 days).
    - [x] Analyze Shift: Invoke `gpt-4o-mini` with a structured prompt to identify current "Top 3 Narrative Drivers" and compare with previous drivers.
    - [x] Cache Result: Store today's narrative in `ResearchCache` with key `narrative_YYYYMMDD` and `source_type="NARRATIVE"`.
- [x] Update tool registry (implicit in Agent definition).

### Verification Criteria
- `get_market_narrative` correctly fetches current index levels.
- Tool retrieves the correct "previous" narrative if it exists in the cache.
- Tool returns a structured string/object with current narrative, top 3 drivers, and the shift direction.
- Successful caching of today's results.
