# Phase 20: Market Narrative & Sentiment Change Agent

## Step 20-02: Trading Calendar Utilities

### Goal
Implement robust logic to determine the previous trading day for historical comparison, accounting for weekends and exchange holidays.

### Context
Simple `timedelta(days=1)` logic fails on weekends and holidays. We need an industry-standard way to look back to the last *actual* session.

### Tasks
- [x] Install `pandas_market_calendars`: `pip install pandas_market_calendars`.
- [x] Create `src/graph/utils/calendar.py`.
- [x] Implement `get_previous_trading_day(target_date: date) -> date`.
- [x] Add unit test `tests/unit/utils/test_calendar.py` to verify logic.

### Verification Criteria
- `get_previous_trading_day` returns correct dates for:
    - Monday -> Previous Friday.
    - Day after holiday -> Day before holiday.
    - Normal Tuesday-Friday -> Previous day.
- Unit tests pass.
