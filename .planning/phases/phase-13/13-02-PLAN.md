# 13-02: History Retrieval & Management APIs

Provide endpoints for the UI to consume the relational conversation history, supporting efficient infinite scroll and user-driven deletion.

## Tasks

<task id="1">
   <name>Implement Cursor-Based Pagination & Backfill</name>
   <files>
     - src/schemas/threads.py
     - src/controllers/threads.py
   </files>
   <action>
     - Update `src/schemas/threads.py` with `HistoryResponse` and `CursorInfo` models.
     - Implement `GET /threads/{id}/history` in `src/controllers/threads.py`.
     - **Backfill Check:** Before querying `chat_messages`, check if relational history exists. If not, trigger the backfill service to promote history from the checkpointer (Redis).
     - Use Keyset Pagination: filter where `ChatMessage.id < cursor` (if provided), `deleted_at IS NULL`, and order by `ChatMessage.id` DESC.
     - Ensure `user_id` filtering for multi-tenancy.
   </action>
   <verify>
     - Verify API returns newest messages first.
     - Verify providing a `cursor` (last ID seen) returns the next page correctly.
     - Verify that history is automatically backfilled for threads that only exist in the checkpointer.
   </verify>
   <done>
     - [ ] Paginated history retrieval and backfill implemented.
   </done>
</task>

<task id="2">
   <name>Implement Soft-Delete Endpoints</name>
   <files>
     - src/controllers/threads.py
   </files>
   <action>
     - Update `DELETE /threads/{id}` to perform a soft-delete (set `deleted_at`).
     - Add `DELETE /threads/{id}/messages/{msg_id}` for individual message removal.
     - Ensure all deletions are user-scoped.
   </action>
   <verify>
     - Verify deleted records are hidden from `GET` endpoints but remain in the database with `deleted_at` set.
   </verify>
   <done>
     - [ ] Soft-delete endpoints functional.
   </done>
</task>

## Success Criteria
- UI can perform infinite scrolling via cursor-based pagination.
- Deleted threads and messages are successfully hidden from API results.
- Cross-user data discovery is blocked (Stealth 404).
