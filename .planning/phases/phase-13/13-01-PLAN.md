# 13-01: Relational History Schema & Dual-Write Logic

Establish the permanent storage foundation for conversation history and implement the background process to promote data from transient agent state to persistent relational tables.

## Tasks

<task id="1">
   <name>Define ChatMessage Model</name>
   <files>
     - src/database/models.py
   </files>
   <action>
     - Add `ChatMessage` model to `src/database/models.py`.
     - Fields: `id` (UUIDv7 PK), `user_id` (FK), `thread_id` (FK), `role` (String: "Human" or "AI"), `content` (Text), `langchain_msg_id` (String, indexed, for idempotency), `created_at` (DateTime), `deleted_at` (DateTime, nullable).
     - Add `deleted_at` to `ChatThread` model for soft-delete support.
     - Add relationships between `User`, `ChatThread`, and `ChatMessage`.
     - Generate and apply Alembic migration.
   </action>
   <verify>
     - `alembic upgrade head` succeeds.
     - `chat_messages` table exists with correct constraints.
   </verify>
   <done>
     - [ ] ChatMessage model defined and migrated.
   </done>
</task>

<task id="2">
   <name>Implement History Sync Service</name>
   <files>
     - src/services/history.py
   </files>
   <action>
     - Create `src/services/history.py`.
     - Implement `sync_conversation_to_postgres(db: AsyncSession, thread_id: str, user_id: str, new_messages: List[BaseMessage])`.
     - Logic: Map `HumanMessage` to "Human" role and `AIMessage` to "AI" role.
     - Use `langchain_msg_id` to prevent duplicates (upsert pattern).
     - Handle soft-delete checks (do not sync to deleted threads).
   </action>
   <verify>
     - Verify sync logic correctly maps roles and persists content via unit tests.
   </verify>
   <done>
     - [ ] History sync service implemented.
   </done>
</task>

<task id="3">
   <name>Integrate with Chat Controller</name>
   <files>
     - src/controllers/chat.py
   </files>
   <action>
     - Update `src/controllers/chat.py`.
     - In `event_generator`, after the LangGraph stream completes, extract the final User/AI exchange.
     - Register `sync_conversation_to_postgres` as a FastAPI `BackgroundTask`.
   </action>
   <verify>
     - Send a chat request and verify two records (Human + AI) appear in the `chat_messages` table.
   </verify>
   <done>
     - [ ] Background sync integrated into chat flow.
   </done>
</task>

## Success Criteria
- Relational message history is automatically populated after every successful chat exchange.
- Duplicate messages are prevented via unique constraint on `langchain_msg_id`.
- System performance is unaffected by background synchronization.
