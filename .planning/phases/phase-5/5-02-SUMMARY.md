# Phase 5 Plan 5-02: Complexity Analysis & Metadata Logic Summary

Implemented complexity analysis scoring and enhanced metadata generation for reports. The summarizer agent now produces structured metadata (title, category, topic, tags) and a linguistic complexity score to enable conditional interrupts.

## Key Changes

### Complexity Service
- Created `src/services/complexity.py` with `evaluate_complexity`.
- **P0 Gating:** Mandatory interrupt (score=100) if action keywords (BUY, SELL, TRADE, INVEST) are detected.
- **P1 Metrics:** Normalized linguistic complexity (0-100) using `textstat.flesch_reading_ease`.

### State Enrichment
- Defined `PendingReport` TypedDict in `src/graph/state.py`.
- Updated `AgentState` to use structured `PendingReport` instead of a generic dictionary.
- Fields added: `title`, `category`, `topic`, `tags`, `complexity_score`.

### Enhanced Summarizer Agent
- Replaced free-text LLM invocation with `with_structured_output` using a Pydantic `SummarizerOutput` model.
- Updated `SUMMARIZER_SYSTEM_PROMPT` to provide instructions for metadata extraction.
- Integrated `evaluate_complexity` into the summarizer flow.
- The agent now ensures the final answer respects the user's language while providing standardized metadata.

## Decisions Made
1. **Structured Output:** Used Pydantic for the summarizer's final output to ensure type safety and consistent metadata across all reports.
2. **Unified Topic Field:** The `topic` field is now generated by the LLM as a high-level subject (e.g., "NVDA" or "Seattle Housing"), augmenting the regex-based `symbol` extraction.
3. **Complexity Normalization:** Mapped Flesch Reading Ease (where 0 is hard) to a Complexity Score (where 100 is hard) for more intuitive threshold logic.
