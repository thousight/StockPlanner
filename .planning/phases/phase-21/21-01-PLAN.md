# Phase 21: Secure Runtime Implementation - Plan 01

This plan focuses on building the foundational security and execution layers for the Python Code Execution Agent, utilizing E2B as the primary sandbox.

## Wave 1: Infrastructure & Security (REQ-500, REQ-502)
<task id="21-01-01" wave="1">
Add `e2b-code-interpreter` to `requirements.txt` and update `src/config.py` to include `E2B_API_KEY`.
<automated>pip install -r requirements.txt</automated>
</task>

<task id="21-01-02" wave="1">
Implement `src/graph/utils/security.py` with an `ASTValidator` class.
- Use `ast.NodeVisitor` to whitelist safe nodes.
- Block access to all double-underscore attributes (`__`).
- Blacklist dangerous modules (`os`, `subprocess`, `sys`, `importlib`, etc.) and functions (`eval`, `exec`, `open`).
<automated>pytest tests/unit/utils/test_security.py</automated>
</task>

<task id="21-01-03" wave="1">
Implement an error sanitization utility in `src/graph/utils/security.py`.
- Function to redact system paths (e.g., `/home/user`) from tracebacks.
- Function to map Python exceptions to structured error codes (`TIMEOUT`, `SECURITY_VIOLATION`, `SYNTAX_ERROR`).
<automated>pytest tests/unit/utils/test_security.py</automated>
</task>

## Wave 2: Core Execution Engine (REQ-511, REQ-521)
<task id="21-01-04" wave="2">
Implement `src/graph/tools/code_executor.py` with the `PythonSandbox` wrapper.
- Use `e2b_code_interpreter.AsyncSandbox`.
- Implement `execute(code, data)` method that injects data as inline JSON.
- Ensure `sandbox.kill()` is called in a `finally` block or via context manager.
- Enforce 5s timeout and capture `stdout`/`stderr`.
<automated>pytest tests/integration/test_code_executor.py</automated>
</task>

<task id="21-01-05" wave="2">
Implement a basic PII filtering utility (REQ-520) for input data.
- Redact obvious email patterns and potential tokens before passing to the sandbox.
<automated>pytest tests/unit/utils/test_security.py</automated>
</task>

## Verification Criteria
- [ ] `ASTValidator` successfully blocks `import os` and `().__class__` style escapes.
- [ ] `PythonSandbox` executes a simple math/pandas script and returns the result from E2B.
- [ ] Failed executions return sanitized error messages without internal file paths.
- [ ] Sandboxes are correctly killed after execution to prevent resource leakage.

## must_haves
- All code must pass `ASTValidator` before being sent to E2B.
- Timeout (5s) must be strictly enforced at the E2B level.
- PII filtering must be applied to the `data` payload before injection.
