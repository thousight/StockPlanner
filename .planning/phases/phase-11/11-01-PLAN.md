# 11-01: Legacy Data Migration

Transition all existing developer/test data to the new primary UUID format to ensure referential integrity before enforcing strict isolation.

## Tasks

<task id="1">
   <name>Create Data Migration Script</name>
   <files>
     - migrations/versions/
   </files>
   <action>
     - Create a manual Alembic migration script.
     - **Pre-check:** Ensure a user with ID `019cb265-1862-76d0-8ebb-10273b096f66` exists in the `users` table; create a dummy record if missing to satisfy foreign key constraints.
     - Target tables: `transactions`, `holdings`, `daily_snapshots`, `chat_threads`, `reports`, `assets`.
     - Update all records where `user_id` is "test-user" or other legacy strings to `019cb265-1862-76d0-8ebb-10273b096f66`.
     - Use `op.execute` for raw SQL updates.
   </action>
   <verify>
     - Run `alembic upgrade head`.
     - Verify no records with "test-user" remain in the specified tables via a quick DB check script.
   </verify>
   <done>
     - [ ] Legacy data migrated to primary UUID.
   </done>
</task>

## Success Criteria
