# 11-02: Context Propagation & Controller Refactor

Move from header-based user identification to secure JWT context using Python `ContextVar` for deep service propagation.

## Tasks

<task id="1">
   <name>Implement ContextVar Propagation</name>
   <files>
     - src/services/auth.py
   </files>
   <action>
     - Define `user_id_ctx: ContextVar[str]` in `src/services/auth.py`.
     - Implement a new dependency `set_user_context(user: User = Depends(get_current_user))` that:
       - Sets `user_id_ctx` with `user.id`.
       - Yields the user.
       - Resets the context in a `finally` block.
   </action>
   <verify>
     - Verify `user_id_ctx.get()` returns the correct ID within an authenticated request.
   </verify>
   <done>
     - [ ] ContextVar propagation implemented.
   </done>
</task>

<task id="2">
   <name>Refactor Controllers for Isolation</name>
   <files>
     - src/controllers/portfolio.py
     - src/controllers/transactions.py
     - src/controllers/threads.py
     - src/controllers/chat.py
     - src/controllers/reports.py
   </files>
   <action>
     - Remove all instances of `x_user_id: str = Header(..., alias="X-User-ID")`.
     - Use `Depends(set_user_context)` to acquire `current_user`.
     - Update all database queries to filter by `current_user.id`.
     - Implement "Stealth 404": ensure any resource access or deletion first checks ownership and returns `404` if user mismatch.
   </action>
   <verify>
     - API documentation (Swagger) should no longer show `X-User-ID` headers.
     - Existing tests should pass when using a valid JWT (will require test refactoring in 11-03).
   </verify>
   <done>
     - [ ] Controllers refactored for authenticated isolation.
   </done>
</task>

## Success Criteria
- `X-User-ID` header is completely removed from the API.
- All protected endpoints use JWT context for multi-tenancy.
- Ownership mismatches return 404.
