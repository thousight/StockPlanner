# Phase 33: Deep News Aggregator - Plan 01

This phase focuses on upgrading the news fetching capabilities of StockPlanner to include structured financial sources, better deduplication, and topic-based clustering.

## Goals
- [ ] Implement `FinancialNewsService` for broader source coverage.
- [ ] Add support for "Deep Research" triggers in the agent graph.
- [ ] Implement article clustering/deduplication based on semantic similarity.

## Tasks
1. **Research & Strategy**: Identify top free financial news RSS/API feeds (Reuters, CNBC, Bloomberg, Yahoo Finance) that can be reliably scraped.
2. **Implementation**:
    - Create `src/services/news_aggregator.py`.
    - Enhance `get_summary` to handle bulk scraping efficiently.
    - Implement simple semantic clustering using LLM or basic vector distance (if available).
3. **Graph Integration**: Add a `DeepResearch` node to the LangGraph mesh for intensive news-gathering tasks.
4. **Validation**: Test with high-volume tickers (AAPL, TSLA, NVDA) and macro keywords.

## Verification Strategy
- **Unit Tests**: Test new aggregation service with mocked HTTP responses.
- **Integration Tests**: Verify that the `DeepResearch` node correctly gathers and synthesizes multiple articles.
- **Performance**: Ensure bulk scraping doesn't exceed LLM token limits or API rate limits.
