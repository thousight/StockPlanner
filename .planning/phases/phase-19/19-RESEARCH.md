# Phase 19 Research: Free Macro Data & Caching

## 1. Free Economic Calendar Source: Financial Modeling Prep (FMP)
- **Endpoint:** `https://financialmodelingprep.com/api/v3/economic_calendar`
- **Authentication:** Requires a free API key passed as an `apikey` query parameter.
- **Parameters:**
  - `from`: Start date (YYYY-MM-DD)
  - `to`: End date (YYYY-MM-DD). Max range is usually 90 days.
- **Response Structure:** Returns a JSON array of events containing:
  - `date`: Release time (UTC).
  - `country`: Two-letter ISO code (e.g., "US").
  - `event`: Name of the indicator.
  - `impact`: Expected volatility ("Low", "Medium", "High").
  - `estimate`: Consensus forecast.
  - `previous`: Prior release value.
  - `actual`: Actual value (if released).
- **Strategy:** Replace the Finnhub client with an FMP client, requesting events for the next 7 days and filtering for `country == "US"` and `impact == "High"`.

## 2. Tool-Level Caching Architecture
- **Objective:** Cache the final Markdown narrative generated by `get_key_macro_indicators` rather than caching individual service responses. This saves LLM tokens and execution time.
- **Cache Schema (ResearchCache):**
  - `source_type`: `ResearchSourceType.MACRO`
  - `key`: Date-based string, e.g., `macro_YYYYMMDD` (e.g., `macro_20260305`). This naturally resets the cache at midnight UTC.
- **Invalidation Strategy:**
  - The tool will check for the existence of the `macro_YYYYMMDD` key for the current UTC day.
  - The `expire_at` for this cache entry should be set to the end of the current UTC day (e.g., `2026-03-05 23:59:59`).
  - *Future Enhancement:* If a high-impact event is detected earlier in the day, the TTL could be shortened to expire exactly after the event time. For simplicity in this phase, a daily cache reset is sufficient.
- **Fallback:** If FMP API fails or returns no data, the `MacroResearcher` retains access to `web_search` to manually find the economic calendar.
