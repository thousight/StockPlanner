# Plan 19-02: Tool-Level Macro Caching

**Goal:** Implement high-performance caching for the final macro narrative report to save LLM tokens and reduce latency.

## Wave 2: Caching Implementation
<task id="19-02-01" wave="2">
Update `src/graph/tools/macro.py`:
- Implement `get_key_macro_indicators` cache check using key: `macro_YYYYMMDD`.
- If cache hit, return directly from `ResearchCache`.
- On cache miss, gather data, format report, and save to cache with `expire_at` set to end-of-day UTC.
</task>

<task id="19-02-02" wave="2">
Implement Cache Override:
- Add support for `refresh_macro` in `kwargs`. If True, bypass cache.
</task>

## Verification Criteria
- [ ] First call to `get_key_macro_indicators` hits the API and saves to DB.
- [ ] Subsequent calls in the same day hit the cache (verified via logs/mock counters).
