# Phase 16 Context: Social & Sentiment

## Overview
This phase implements a unified sentiment and social media analysis engine. It consolidates scraping utilities, introduces numerical sentiment scoring with LLM-driven rationale, and integrates a direct X/Twitter API connection for real-time social sentiment.

## Implementation Decisions

### 1. Tooling & Scraping Consolidation
- **Unified Utility:** Create `src/graph/utils/scraping.py` to handle all HTML fetching and cleaning.
- **User-Agent:** Pass the `User-Agent` string down from the initial request header (FastAPI -> Graph -> Tools -> Utils).
- **Parsing:** Apply `readability-lxml` to all web content (including SEC filings) to extract the primary text body.
- **Error Handling:** Fetch failures (403, 404, timeouts) return an empty string. Failed fetches are **not** cached.
- **Unified Cache:** Consolidate `NewsCache` and `SECCache` into a single `ResearchCache` table with a `source_type` (SEC, NEWS, SOCIAL) column.

### 2. Sentiment Engine
- **Granularity:** Use a numerical score from **-1.0 to 1.0**.
- **Rationale:** Every sentiment score MUST include a concise "Reason" generated by the LLM.
- **Persistence:** The `ResearchCache` will store both the `sentiment_score` and `sentiment_reason` to minimize redundant LLM calls.
- **Aggregation Logic:** The Analyst agent will both **average** the scores for a general pulse and explicitly **highlight divergence** if high-confidence sources (SEC) conflict with high-volume sources (Social).

### 3. Social Media Strategy
- **X/Twitter API Integration:** Implement a dedicated service (`src/services/social.py`) for the official X API. 
    - Focus on fetching recent high-engagement tweets for specific tickers/cashtags (e.g., `$AAPL`).
    - Use async clients for non-blocking sentiment gathering.
- **Reddit/Web Scraping:** Continue using hardcoded `site:` operators in DuckDuckGo searches for Reddit (r/wallstreetbets) and StockTwits.
- **Interaction Model:** "Social Sentiment" will have its own dedicated **interaction bubble** in the agent output.
- **Noise Filter:** Social insights are only mentioned if they cross a "High Intensity" threshold.

### 4. Temporal Dynamics
- **Social Expiry:** Social media results (especially from X) have a strict **24-hour TTL**. 
- **Precedence:** In any conflict, **Fundamentals (SEC/10-K)** always take priority in the final investment thesis.
- **Shift Detection:** The system will report **"Sentiment Shifts"** if the user asks about the same ticker across multiple sessions within the same day.

## Code Context
- **Services:** New `src/services/social.py` for X API integration.
- **Utilities:** Unified functions in `src/graph/utils/scraping.py`.
- **Database:** Update `src/database/models.py` for the unified `ResearchCache` and sentiment columns.
- **Agents:** `ResearchAgent` spawns "Social Sentiment" bubbles; `AnalystAgent` weighs these scores.

## Deferred Ideas (Out of Scope)
- Social Media Volume as a primary metric (focusing on Sentiment/Vibe only).
- Sentiment analysis for non-English social media posts (prioritizing English for now).
