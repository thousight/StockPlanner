# 14-02: Protocol-Compliant Generator

Refactor the SSE generator to yield standard tags and JSON payloads, supporting multiple stream modes and nested subgraph propagation.

## Tasks

<task id="1">
   <name>Refactor event_generator for Protocol</name>
   <files>
     - src/controllers/threads.py
   </files>
   <action>
     - Implement `langgraph_event_generator` in `src/controllers/threads.py`.
     - Logic:
       - Initialize graph with `thread_id` and `stream_mode` requested.
       - Use `graph.astream(input, config, stream_mode=stream_mode)`.
       - Iterate through yielded events.
       - For each event, yield `event: <mode>
data: <json_payload>

`.
       - Map `messages` mode: emit `AIMessageChunk` data.
       - Map `metadata` mode: emit run/node metadata.
       - Map `values` mode: emit full state.
       - Emit `event: end
data: {}

` at completion.
   </action>
   <verify>
     - Use a tool (like `curl`) to verify the raw stream contains `event:` lines and properly formatted `data: [...]` JSON arrays.
   </verify>
   <done>
     - [ ] Protocol-compliant generator implemented.
   </done>
</task>

## Success Criteria
- Stream yields `event: messages` with standard `AIMessageChunk` JSON.
- Stream explicitly ends with `event: end`.
- Multiple simultaneous stream modes (messages + metadata) work correctly.
- Error events are yielded using the `error` event tag.
