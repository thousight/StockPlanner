# Phase 24 Research: Monitoring & Auditing

## 1. Structured JSON Audit Logging
We will implement a dedicated audit trail for all code execution events using `structlog` or a configured standard `logging` JSON formatter.

### Logging Strategy:
- **Logger Name:** `audit.sandbox`
- **Format:** JSON (key-value pairs).
- **Captured Fields:**
    - `timestamp`: ISO 8601.
    - `thread_id`: To correlate with the conversation.
    - `correlation_id`: To correlate with the specific API request.
    - `code`: The full Python script generated by the agent.
    - `input_data`: The full JSON payload injected.
    - `status`: `SUCCESS`, `FAILURE`, `TIMEOUT`, `SECURITY_VIOLATION`.
    - `duration_ms`: Total execution time.
    - `output_summary`: Length of stdout and number of results returned.

## 2. E2B Concurrency Management
E2B enforces a limit of 100 concurrent sandboxes for Pro accounts. To ensure stability and avoid hard crashes, we will implement local concurrency control.

### Implementation Pattern:
- **Async Semaphore:** Use a global `asyncio.Semaphore(90)` to cap concurrent sandbox creation.
- **Rate Limit Handling:** Wrap `Sandbox.create()` in a retry loop using `tenacity` to handle transient `RateLimitException` (429) errors.
- **Resource Cleanup:** Enforce `async with` or strict `try...finally` to ensure `sandbox.kill()` is always called, immediately freeing the concurrency slot.

## 3. Deep Output Safeguarding (REQ-520)
Building on the Phase 21 input filter, we will implement a post-execution scan of the results.

### Scanning Logic:
1. **Raw Stdout:** Scan the entire `stdout` string for patterns matching emails, phone numbers, or tokens.
2. **Structured Results:** Recursively scan the `results` list (returned objects) for string values that match PII patterns.
3. **Redaction:** Apply the same `strip_pii` utility to redact found patterns *before* the results reach the agent.

---

## 4. Stress Testing Scenarios
A dedicated test suite `tests/stress/test_sandbox_stress.py` will be created to verify limits.

### Scenarios:
- **Infinite Loop:** `while True: pass` (Verify 5s timeout kills the sandbox).
- **Memory Bomb:** `[0] * 10**8` (Verify behavior near 50MB limit).
- **Concurrency Spike:** Spawning 50 sandboxes simultaneously (Verify Semaphore logic).
- **Data Bloat:** Passing a 5MB JSON payload (Verify serialization overhead).
