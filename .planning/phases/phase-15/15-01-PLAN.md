## Verification Criteria
- [ ] `SECCache` table exists in the database.
- [ ] `extract_section` correctly parses Item 1A from the local 10-K fixture.
- [ ] `get_latest_filing_urls` returns valid SEC.gov URLs for a given ticker.
- [ ] Unit tests for parsing logic are passing.

## must_haves
- SEC extraction logic must NOT hit SEC.gov without a valid `User-Agent`.
- Parsing must be robust enough to handle common 10-K/Q HTML structures.
- Data must be cached to minimize external requests and latency.
