# Plan 18-01: Authoritative Macro Services

**Goal:** Establish the backend services for fetching hard economic data from FRED and the economic calendar from Finnhub.

## Wave 1: Infrastructure & Services
<task id="18-01-01" wave="1">
Update `src/config.py` to include `FRED_API_KEY` and `FINNHUB_API_KEY`.
Update `requirements.txt` if needed (though we'll use `httpx` for direct REST calls).
<automated>pytest tests/unit/test_config.py</automated>
</task>

<task id="18-01-02" wave="1">
Create `src/services/macro.py` and implement the `FEDService`.
- Implement `get_series_data(series_id, limit=6)` to fetch the last 6 months/entries.
- Map friendly names to Series IDs: `GDP`, `CPI`, `FEDFUNDS`, `PAYEMS`, `DXY`.
- Use async `httpx` for all calls.
<automated>pytest tests/unit/services/test_macro.py</automated>
</task>

<task id="18-01-03" wave="1">
Implement the `EconomicCalendarService` in `src/services/macro.py`.
- Implement `get_upcoming_events(days=7)` using the Finnhub `/calendar/economic` endpoint.
- Filter for US region and High Impact events.
<automated>pytest tests/unit/services/test_macro.py</automated>
</task>

## Verification Criteria
- [ ] `FEDService` successfully fetches GDP and CPI data.
- [ ] `EconomicCalendarService` returns a list of high-impact US events.
- [ ] Both services handle API errors gracefully with empty fallbacks.
