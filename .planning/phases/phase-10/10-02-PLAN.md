# 10-02: Refresh Token Rotation & Session Management

Implement secure token rotation, sign-out, and administrative session revocation.

## Tasks

<task id="1">
   <name>Implement Token Rotation & Refresh</name>
   <files>
     - src/schemas/auth.py
     - src/services/auth.py
     - src/controllers/auth.py
   </files>
   <action>
     - Update `src/schemas/auth.py`:
       - Add `TokenRefreshRequest` (refresh_token).
     - Update `src/services/auth.py`:
       - Implement `rotate_refresh_token(db: AsyncSession, refresh_token: str, user_agent: str) -> (str, str)`:
         - Verify signature and expiry.
         - Look up `token_hash` in database.
         - **Theft Detection:** If token already rotated > 10s ago, call `revoke_all_user_sessions`.
         - **Grace Period:** If rotated < 10s ago, return the *new* token from that rotation.
         - Else: Create new access + refresh tokens, link `parent_token_id`, and mark old one as rotated.
     - Update `src/controllers/auth.py`:
       - Add `POST /refresh`: Accepts `TokenRefreshRequest`, returns new `TokenResponse`.
   </action>
   <verify>
     - Verify that using a refresh token twice within 10s works (returns same new pair).
     - Verify that using a refresh token twice after 10s revokes everything.
   </verify>
   <done>
     - [ ] Token rotation and refresh implemented.
   </done>
</task>

<task id="2">
   <name>Implement Session Revocation</name>
   <files>
     - src/services/auth.py
     - src/controllers/auth.py
   </files>
   <action>
     - Update `src/services/auth.py`:
       - Implement `revoke_refresh_token(db: AsyncSession, refresh_token: str)`: Deletes the specific token.
       - Implement `revoke_all_user_sessions(db: AsyncSession, user_id: str)`: Deletes all tokens for the user.
     - Update `src/controllers/auth.py`:
       - Add `POST /signout`: Accepts current refresh token and revokes it.
       - Add `POST /revoke-all`: Revokes all sessions for the authenticated user.
   </action>
   <verify>
     - Verify that sign-out makes the refresh token unusable.
     - Verify that revoke-all clears all entries in `refresh_tokens` for that user.
   </verify>
   <done>
     - [ ] Session revocation endpoints implemented.
   </done>
</task>

## Success Criteria
- Refresh token rotation works with 10s grace period.
- Theft detection successfully invalidates all sessions.
- Sign-out and Revoke-All functional and verified.
