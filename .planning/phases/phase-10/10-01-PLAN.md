# 10-01: JWT Core & Sign-In API

Establish the foundation for JWT management and implement the sign-in flow.

## Tasks

<task id="1">
   <name>Setup Configuration & Dependencies</name>
   <files>
     - requirements.txt
     - src/config.py
   </files>
   <action>
     - Add `PyJWT` to `requirements.txt`.
     - Update `src/config.py`:
       - Add `JWT_SECRET_KEY` (str).
       - Add `JWT_ALGORITHM` (default "HS256").
       - Add `ACCESS_TOKEN_EXPIRE_MINUTES` (10).
       - Add `REFRESH_TOKEN_EXPIRE_MINUTES` (10).
   </action>
   <verify>
     - Check `src/config.py` for new settings.
   </verify>
   <done>
     - [ ] Configuration and dependencies updated.
   </done>
</task>

<task id="2">
   <name>Define RefreshToken Model</name>
   <files>
     - src/database/models.py
   </files>
   <action>
     - Add `RefreshToken` model to `src/database/models.py`:
       - `id`: UUID primary key.
       - `user_id`: ForeignKey to `users.id`.
       - `token_hash`: String (SHA-256 hash of the JTI).
       - `user_agent`: String (nullable).
       - `parent_token_id`: ForeignKey to `refresh_tokens.id` (nullable, for rotation chains).
       - `rotated_at`: DateTime (nullable).
       - `expires_at`: DateTime.
       - `created_at`: DateTime.
     - Generate and apply Alembic migration.
   </action>
   <verify>
     - `alembic upgrade head` succeeds and `refresh_tokens` table exists.
   </verify>
   <done>
     - [ ] RefreshToken model defined and migrated.
   </done>
</task>

<task id="3">
   <name>Implement JWT Service & Sign-In</name>
   <files>
     - src/services/auth.py
     - src/schemas/auth.py
     - src/controllers/auth.py
   </files>
   <action>
     - Update `src/schemas/auth.py`:
       - Add `UserSignIn` (email, password).
       - Add `TokenResponse` (access_token, refresh_token, token_type).
     - Update `src/services/auth.py`:
       - Implement `create_access_token(user_id: str) -> str`.
       - Implement `create_refresh_token(db: AsyncSession, user_id: str, user_agent: str) -> str`.
       - Implement `authenticate_user(db: AsyncSession, data: UserSignIn) -> User`.
     - Update `src/controllers/auth.py`:
       - Add `POST /signin`: Validates credentials, issues tokens, returns `TokenResponse`.
   </action>
   <verify>
     - `POST /auth/signin` returns valid JWTs for registered users.
   </verify>
   <done>
     - [ ] Sign-In API and JWT service implemented.
   </done>
</task>

## Success Criteria
- Valid JWTs (Access & Refresh) issued upon successful sign-in.
- Passwords verified with Argon2id.
- Sign-in fails with 401 for incorrect credentials.
- Refresh tokens persisted in the database.
